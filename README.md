# SecretKey

A deterministic, stateless password generator, recovery and sharing tool built as a small Visual Studio solution (target: .NET 10).  
SecretKey derives passwords feom a 1PASSWORD style .csv file using a secret key and a date code. There is no password database required.

The secret key is a 32-byte master key (hex-encoded) that you keep private and is stored in a json configuration file.

The date code is a 4-digit number (YYMM) that only you know. The passwords are derived from the input .csv file using something you have (the secret key) and something you know (the date code). 

This allows you to generate the same passwords consistently without storing them anywhere. 

The generator operates on a CSV file exported from 1Password (or similar) and produces a new CSV with the same entries but with the passwords replaced by the generated ones.

The input CSV is expected to have a header row and the following columns: `title`, `username`, `password`, `url`, `notes`. 

The input .csv file is normalized and the old password column is removed immedialy to prevent password leaks. 

You can use the date code to generate new passwords periodically (e.g., monthly) while keeping the same secret key.

The generator produces consistent passwords based on the input csv, the secret key and datecode allowing you to manage your passwords without storing them.

The passwords can be recovered at any time by running the generator with the same secret key and date code, making it a stateless solution.

The passwords can be shared securely by sharing the secret key and date code with trusted parties, without sharing the actual passwords.

For sharing, I can USPS mail a flash drive with the normailized .csv file and config file containing the secret key. 

This provides an air gap from the internet so the password can be generated by also sharing the date code.
